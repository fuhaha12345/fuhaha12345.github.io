闪电贷是 DeFi 中一种无需抵押的原子借贷机制：借款、使用资金和还款必须在同一笔交易中完成。闪电贷本身不是漏洞，而是一种提高资金效率的金融工具；真正的风险在于协议把“单笔交易内可操纵的状态”误认为可靠输入。

<!--more-->

## 工作原理

一次闪电贷通常包含以下步骤：

```text
借入资金 → 执行业务逻辑 → 获利 → 归还本金和手续费
```

借款只存在于当前交易的执行期间。若交易结束前没有足额归还，合约会触发 `revert`，借款、交易和状态修改会一起回滚。因此，借款方可以在零抵押的情况下临时获得大量流动性，贷款方则依靠 EVM 的原子性保证资金安全。

一个简化的实现如下：

```solidity
function flashLoan(uint256 amount) external {
    uint256 balanceBefore = token.balanceOf(address(this));

    // 1. 转出贷款
    token.transfer(msg.sender, amount);

    // 2. 回调借款方，在同一笔交易内执行其逻辑
    IReceiver(msg.sender).execute();

    // 3. 确认贷款已归还
    require(
        token.balanceOf(address(this)) >= balanceBefore,
        "Loan not repaid"
    );
}
```

实际协议还会校验回调返回值、收取手续费，并使用安全的代币转账库。上面的代码只用于说明原子借贷流程。

## 为什么闪电贷会放大漏洞

闪电贷降低了攻击者获取大额短期资本的门槛。若抵押率、投票权、兑换价格、奖励份额或清算条件直接读取同一交易中可改变的余额，攻击者就能临时借入巨量资产，操纵状态并调用敏感函数获利。

通用攻击流程如下：

1. 通过闪电贷借入目标资产。
2. 在同一交易中改变 AMM 储备、抵押余额、治理权重或奖励份额。
3. 调用依赖该瞬时状态的敏感函数。
4. 将市场状态恢复或兑换资产。
5. 归还闪电贷及手续费，保留差额。

## 常见攻击类型

### 价格预言机与 AMM 现货价格操纵

AMM 的即时价格由池中资产储备比例决定。攻击者可以借入大量资金并进行大额交易，短暂改变储备比例；如果其他协议直接把该池的现货价格作为抵押估值或清算依据，就会读取到被操纵的价格。

```text
闪电贷借入大量资产
        ↓
大额交易改变 AMM 储备比例
        ↓
目标资产现货价格被暂时推高或压低
        ↓
协议按错误价格执行借贷、兑换或清算
        ↓
反向交易并归还闪电贷
```

下面的估值逻辑依赖单点现货价格，因而容易受到操纵：

```solidity
function collateralValue(address user) public view returns (uint256) {
    return balances[user] * pool.getSpotPrice();
}
```

### 治理攻击

如果治理权直接取决于当前代币余额，攻击者可能临时借入大量治理代币，在同一交易或极短时间内获得足够投票权，通过恶意提案后再归还代币。

```text
借入治理代币 → 获得临时投票权 → 通过恶意提案 → 执行提案 → 归还代币
```

### 奖励机制攻击

如果奖励按照领取时的瞬时余额计算，攻击者可以借入代币、临时提高余额或资金池份额、领取不成比例的奖励，然后立即退出并归还贷款。根本问题是协议使用短期余额决定长期收益。

```text
借入代币 → 临时提高余额或份额 → 领取奖励 → 退出头寸 → 归还代币
```

## 防御方法

- **价格来源**：使用时间加权平均价格（TWAP）或经过验证的多源预言机，不直接依赖单个 AMM 的即时现货价格。
- **操纵成本**：为价格偏差、交易滑点、借贷额度和清算参数设置合理上限及异常检测。
- **治理快照**：按历史区块的余额计算投票权，并为提案、投票和执行设置延迟，避免临时借入代币取得治理权。
- **奖励记账**：按照持仓时长、周期快照或累计积分分配奖励，避免用领取时的瞬时余额决定收益。
- **状态隔离**：重要参数变更采用时间锁，避免在同一交易内完成“操纵状态—调用敏感函数—恢复状态”的闭环。
- **组合测试**：在测试和审计中假设攻击者能在一笔交易内获得近乎无限的短期流动性，验证关键状态是否可被原子操纵。

## 总结

闪电贷提供的是资金能力，而不是漏洞本身。协议设计时不应信任可在单笔交易内显著变化的余额、价格或权重；只要敏感决策依赖这类瞬时状态，就应使用时间维度、独立数据源、额度限制和延迟机制降低可操纵性。
